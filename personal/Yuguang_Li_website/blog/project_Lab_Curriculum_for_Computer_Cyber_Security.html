Lab Curriculum for Computer Cyber Security

The resources used and experience in our Lab curriculum will be helpful to those who are considering learning the basic cybersecurity knowledge or trying the simple penetration testings.

Introduction

In this blog, I will not talk about much about the course contents for the reason that the graduate course may still use these materials in the future, but I would like to mainly introduce the environment and some topics we used in our Lab curriculum that, I think, could offer some hints to the people who are interested in exploring the cybersecurity world or just plan to try some penetration testings for pleasure.

Environment

The Environment setup constructed with the following desiderata in mind:

  • Simplicity: an introductory tutorial in cybersecurity does not need a complicated environment to achieve its objectives.
  • Security: keep the target (vulnerable) machine off the public Internet; minimize possible effects of the actions on your intranet.
  • Resource efficiency: minimize resource use
  • Reuse: reuse existing components / software as much as possible.

The optimal solution for the beginners' environment is seem to use virtual networks, in which you have both attacker and victim in your own preset virtual networking for security issue. This is easy for you to perform some pen-testings as well as understand where and why the victim is vulnerable.

A workstation with 4 Gb RAM runs VMWare Workstation(Windows/Linux)/VMWare Fusion(OS X). Three virtual machines are used:

  1. The attacker is represented by a Kali Linux virtual machine. This distribution is based on Debian Linux, includes a multitude of security audit and penetration tools (including all the tools introduced in the below topics), and is officially available for download as a VMWare image that boots into a desktop environment. Therefore, Kali is easily accessible even by beginners who are new to Linux.
  2. The target is represented by a Metasploitable 2 virtual machine. This image contains a number of services with known exploits (ftp, http, and so on), insecure settings, and three vulnerable web applications. This VM is also officially available as a VMWare image, and has low memory requirements since it does not offer a graphical user interface.
  3. One of the topics requires a third ”zombie” machine. Almost any virtual machine that has networking support would do; we have chosen a FreeBSD 6.0 image for its low system requirements and for some diversity in the virtual environment (not 100% Linux).A VMWare image for this machine is provided by Simon

Some topics

If you are interested in learning cybersecurity, you could follow the topics I list below as your learning curve. It is pity that I could not offer the contents about all the topics. However, these topics provide you some guidances about the concepts in cybersecurity and the handy tools.

  • Introduction

    Install VMWare (the virtual environment) and the virtual machines used above, which guides a beginner through the installation and configuration process, as well as learn some standard Linux tools, shells and basic networking concepts.

  • Law and Ethics

    Evaluate various actions in context of applicable law and ethics standards of relevant bodies.(This is very important!!)

  • Search Engine Hacking and Social Engineering

    Investigate an organization and discover relevant organizational and personal information using a search engine(such as your friends), illustrating the amount of publicly available sensitive information that can routinely be found on the Internet. Wiki about Social Engineering.

  • Network Utilities and Scripting

    Explore various uses of common versatile network utilities (such as netcat and wget) and do some scripting: in a shell script (utilizing the learned network utilities) and, on a lower level, by interacting with TCP/IP sockets directly from the Python scripting language.

  • Network Attacks

    Search for the network attack tools included with Kali Linux. Specifically, ways to mount ARP and DNS spoofing attacks on a network. Learn "Nmap" scan uzing the “zombie” technique.

  • Metasploit

    Using the "Metasploit framework" to perform several exploits on Metasploitable’s services that are known to be vulnerable, mount different denial-of-service attacks on a web server running on the target VM, and explain why only some of the attacks are successful.

  • Password Cracking

    Introdcue an off-line password cracking tool "John the Ripper" and to on-line tool "Hydra" that can be used to guess passwords to a website requiring authentication. You could use the on-line tool to find valid credentials to a different web application supplied with Metasploitable, and to crack Metasploitable’s password database that they would download after compromising the target machine.

  • Intrusion Detection

    Learn popular intrusion detection tool "snort", investigating varous modes of its operation and developping a custom filter that detects a previously studied Metasploit exploit attempt on a specific service.

    • Conclusion

    This blog is only used for the very beginner without much networking skills to know about cybersecurity and pentestings. In additon, the well-structured virtual envrionment is pretty friendly for beginner to play around. Although only some topics are shown without details, it gives you some hints about what to learn and what to read.
